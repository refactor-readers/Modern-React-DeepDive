# 들어가며

## 리액트는 왜 많은 개발자들이 선호하는 라이브러리가 되었을까?

### 명시적인 상태 변경

- 리액트는 단방향 바인딩만 지원합니다.
  - 단방향 바인딩이란 말 그대로 데이터의 흐름이 한쪽으로만 흘러가는 것 입니다.
  - 이와 반대되는 용어는 Angular의 양방향 바인딩입니다. 양방향으로 바인딩되면 뷰의 변화가 컴포넌트에 영향을 미칠 수도, 반대로 컴포넌트의 상태가 변경되면 뷰의 상태도 변할 수 있습니다.
  - 양방향 바인딩은 단방향이 제공할 수 없는 편리함을 제공하지만, 코드의 규모가 커질수록 상태의 변화가 무엇으로 일어날지 파악하기 어려워집니다.
- 리액트의 상태 변화는 `단방향`으로, 그리고 `명시적`으로 이루어집니다. 상태가 변화했다면 그 상태 변화를 명시적으로 일으키는 함수만 찾으면 되는것이죠.
  - 양방향 바인딩이 항상 나쁘다는 의미는 아닙니다.
  - 단방향 바인딩은 항상 변화를 감지하고 업데이트하는 코드를 매번 작성해야 하며, 이에 따라 코드량이 증가하는 등의 단점이 있습니다.

### JSX(JavaScript XML)

- 리액트는 HTML에 자바스크립트 문법을 더한 JSX를 사용하는데, 고유의 몇가지 특장만 이해한다면 손쉽게 JSX코드를 구현할 수 있습니다.

### 비교적 배우기 쉽고 간결함

- HTML과 자바스크립트에 대해서 어느 정도만 알고 있다면 빠르고 쉽게 컴포넌트와 웹페이즈를 만들 수 있습니다.
  - 구러나 개발 초기에는 쉽지만, 이를 깊게 이해하고 성능최적화 등의 작업은 상대적으로 어려운 축에 속합니다.
  - 다른 라이브러리와 비교했을 때 자유롭게 다루기까지 드는 시간과 노력은 다른 모듈보다 비교적 난이도가 있는 편 입니다.
