# 15.1 리액트 프로젝트를 시작할 때 고려해야 할 사항

## 15.1.1 유지보수 중인 서비스라면 리액트 버전을 …

- 추후에 발생할 업데이트를 대비해 최소한의 버전업이 필요함
- 호환성 문제나 신규 기능 출시 등 업데이트 노트를 확인하고 업데이트 해보자

## 15.1.2 인터넷 익스플로러 11 지원..

- 구형 브라우저를 지원해야 하는 경우 라이브러리 설치에 각별히 주의하자

## 15.1.3 서버사이드 렌더링 애플리케이션을 우선적으로 고려한다

- 모바일 기기 성능은 사용자마다 차이가 큼
- 자바스크립트 코드 실행 속도에 의존하면 사용자 경험의 갭이 커짐
- VOC는 보통 극단적인 사용자에게 발생함
- 성능을 우선하는게 좋음

## 15.1.4 상태관리 라이브러리는 꼭 필요할때만

- Context API 정도로 커버 가능하면 굳이 설치할 필요가..
- 꼭 특정 스택을 고집할 필요는 없음

## 15.1.5 리액트 의존성 라이브러리 설치 조심

- 리액트 의존적인 라이브러리(`react-**`) 설치 시 리액트와 호환되는 버전인지 확인이 필요

# 15.2 언젠가 사라질 수도 있는

## 15.2.1 리액트는 그래서 정말 완벽한 라이브러리인가?

- 가장 널리 쓰이는 프론트엔드 스택, 하지만 반대의견도 많음
- 클래스형 → 함수형 마이그레이션에서의 혼란..
  > 클래스형이 더 어려워요..
- 너무나 많은 옵션으로 인한 혼란

## 15.2.2 오픈소스 생태계의 명과 암

- 라이선스 문제로 오픈소스를 사용한 제품을 한순간에 잃을 수 있음

## 15.2.3 제이쿼리, 앵귤러, 리액트 … 그 다음은?

- 프론트엔드 단독 프레임워크가 증가하는 추세
  > 이제는 진짜 서버 없어도 될 것 같아요
- 리액트는 여전히 굳건한 것 같음
- 다음은.. 스벨트..?

## 15.2.4 웹 개발자로서 가져야 할 유연한 자세

- HTML, CSS, Javascript 기본기를 잘 다져야 함
- 느린 자바스크립트를 대체할 방안으로 웹어셈블리가 뜨고있음
  - Docker보다 용량이 작고 실행속도가 🐶빠름
  - WASI 0.3부터 모듈화도 된다고 함
  - AI 출시로 고성능 컴퓨팅을 위한 수요가 많이 늘어났다고 함
  - 보안쪽에서도 뛰어나다고 하네요(Sandboxing)
- 기본이 되는 자바스크립트를 더 깊게 공부하자
