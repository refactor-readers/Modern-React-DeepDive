# 6.3 리액트 개발 도구 활용

개발자 도구에서 리액트 전용 도구인 Components, Profiler를 사용할 수 있음

디버깅 페이지([Netflix](https://www.netflix.com))

## 6.3.1 컴포넌트

- Component 탭에서는 리액트 컴포넌트 트리를 확인할 수 있음
- 단순 구조와 더불어 props, hook등 다양한 정보를 보여줌

### 컴포넌트 트리

- 기명함수의 경우 함수명(컴포넌트명)을 그대로 보여줌
- 익명함수의 경우 Anonymous로 표시됨
- `displayName`을 활용하면 원하는 명칭으로 개발자도구에 표시할 수 있음
    
    ```tsx
    Component.displayName = "컴포넌트이름"
    // 개발자 도구에 "컴포넌트이름" 그대로 표시됨
    ```
    

### 컴포넌트명과 props

- 컴포넌트를 클릭하면 해당 컴포넌트에 대한 자세한 정보를 확인할 수 있음
- 눈 아이콘: Element탭의 해당 요소로 이동함
- 벌레 아이콘: 콘솔에 해당 컴포넌트 정보가 출력됨
- 소스코드 아이콘: 해당 컴포넌트의 소스코드를 확인할 수 있음

## 6.3.2 프로파일러

- 리액트가 렌더링하는 과정을 확인할 수 있음
- 렌더링 횟수나 시간 등을 확인할 수 있음
- 프로덕션 빌드에서는 사용할 수 없음

### 설정 변경하기

톱니 버튼을 누르면 디버깅을 위한 몇가지 설정을 할 수 있음

1. Highlight updates when components render: 렌더링 될 때 마다 해당 컴포넌트에 하이라이트 표시
2. Hide logs during second render in Strict Mode: strict mode에서 실행되는 경우 두 번 돌아가는걸 방지해서 console.log 등 출력을 한번한 찍히게 할 수 있음
3. Record why each component rendered while profiling: 포르파일링 도중 무엇 때문에 컴포넌트가 렌더링 됐는지 기록함

### 프로파일링

렌더링 과정에 어떤일이 벌어지는지 알 수 있음

#### 도구 조작법

1. Start Profiling
    - 프로파일링 시작/중단
2. Reload and Start Profiling
    - 새로고침 후 프로파일 시작
3. Stop Profiling
    - 프로파일링 중단, 기록 모두 삭제됨
4. Load Profile / Save Profile
    - 프로파일링 결과 저장/불러오기
    - JSON 형태로 다운로드되고, 다시 불러올 수 있음

#### Flamegraph

- 렌더 커밋 별 작업 내용을 보여줌
- 메모이제이션의 작동 여부, 의도한 렌더링이 잘 이루어지고 있는지 등을 확인하는데 도움이 됨

#### Ranked

- 렌더링에 소요된 시간 순으로 나열한 차트
- 실제 렌더링이 발생한 컴포넌트만 보여줌

#### 타임라인

- 시간이 지남에 따라 컴포넌트에서 발생하는 일을 확인할 수 있음
- 18+ 환경에서만 확인할 수 있음
