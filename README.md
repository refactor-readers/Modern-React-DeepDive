## ✨ 스터디 목표

- 꾸준한 리딩 & 페이스 유지하기
- 내 생각을 짧고 명확하게 말/글로 정리하는 연습
- 다양한 시선/해석 나누기
- 기술 면접에서 외워서 말하는 게 아니라 “이해 후” 말을 할 수 있게
- 실무에서도 바로 연결 가능한 React 지식 습득하기

## 🗓 진행 방식

| 항목      | 내용                                                                                                 |
| --------- | ---------------------------------------------------------------------------------------------------- |
| 진행 요일 | 매주 수요일 22:00 (오후 10시)                                                                        |
| 정리 방식 | 각 챕터 읽고 → Pull Request 제출                                                                     |
| 발표 방식 | 랜덤 발표 2명 선정 (희망자가 있으면 우선)                                                            |
| 상호작용  | PR 보고 이모지 / 댓글 / 잡담 / 질문 모두 OK                                                          |
| 정리 포맷 | Markdown 기반 <a href="https://gist.github.com/ihoneymon/652be052a0727ad59601">(**작성법 참고**</a>) |

## 🤝 규칙

- 해당 주 챕터는 발표 전까지 PR로 제출해주세요.
- 너무 길게 작성하기보단 핵심 위주로 정리해주세요!
- 질문은 많을수록 좋습니다.
  - WHAT / HOW / WHY 방식 환영합니다.
  - 퀴즈 형태나 토론을 유도하는 질문도 모두 괜찮습니다.

## 🔧 진행 방법

1. organization repo clone
2. `main`에서 `${아이디}` 브랜치 생성
3. 본인 이름 폴더 생성
4. 챕터별 정리본 업로드 (markdown)
5. `main` 브랜치로 Pull Request
