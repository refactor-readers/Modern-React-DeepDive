# 12장. 모든 웹 개발자가 관심을 가져야 할 핵심 웹 지표
구글이 제시한 사용자 경험(UX)을 측정하는 Core Web Vitals에 대한 내용입니다.

## 12.1 웹사이트와 성능
웹사이트 성능은 사용자 이탈률과 전환율에 직접적인 영향을 미칩니다. 로딩이 1초 늦어질 때마다 전환율이 크게 하락한다는 연구 결과가 있습니다.

## 12.2 핵심 웹 지표란?
구글이 꼽은 세 가지 핵심 지표는 LCP(로딩 성능), FID(상호작용), CLS(시각적 안정성)입니다.

## 12.3 최대 콘텐츠풀 페인트 (LCP)
### 12.3.1 정의
페이지 로딩 시작 후 뷰포트 내에서 가장 큰 이미지나 텍스트 블록이 렌더링되는 데 걸리는 시간입니다.

### 12.3.4 기준 점수
좋음: 2.5초 이내

보통: 4.0초 이내.

### 12.3.5 개선 방안
가능하면 해당 영역을 텍스트로 구현하는 것이 가장 빠릅니다.

이미지 로딩 최적화: <img> 태그는 프리로드 스캐너에 의해 빠르게 발견되지만, CSS background-image는 늦게 발견되므로 중요 리소스에는 피해야 합니다.

LCP 요소에는 loading="lazy" 속성을 사용하지 마세요.

## 12.4 최초 입력 지연 (FID)
### 12.4.1 정의
사용자가 페이지와 처음 상호작용(클릭, 버튼 탭 등)했을 때부터 브라우저가 해당 이벤트 핸들러 처리를 시작하기까지 걸리는 시간입니다.

### 12.4.2 의미
화면은 그려졌지만 자바스크립트 실행하느라 바빠서 클릭해도 반응이 없는 상태를 측정합니다.

### 12.4.4 기준 점수

좋음: 100ms 이내.

### 12.4.5 개선 방안
메인 스레드를 차단하는 긴 작업(Long Task)을 분리하거나 최소화해야 합니다.

타사 스크립트 실행을 지연하거나 필요할 때만 불러옵니다.

## 12.5 누적 레이아웃 이동 (CLS)
### 12.5.1 정의
페이지 생명주기 동안 발생하는 예기치 않은 레이아웃 이동 점수의 합계입니다.

### 12.5.2 의미
사용자가 읽고 있던 글이 갑자기 밀려나거나, 엉뚱한 버튼을 클릭하게 만드는 시각적 불안정성을 측정합니다.

### 12.5.4 기준 점수
좋음: 0.1 이하.

### 12.5.5 개선 방안
이미지나 동적 요소의 크기(width, height)를 미리 지정하여 공간을 확보합니다.

폰트 로딩 시 FOUT/FOIT를 방지하기 위해 font-display: optional 등을 활용합니다.
