# 7.1 크롬 개발자 도구란?

크롬 개발자 도구는 웹페이지에서 일어나는 거의 모든 일을 확인할 수 있는 강력한 도구
웹사이트 디버깅 시에는 브라우저 확장 프로그램에 의한 간섭을 피하기 위해 **시크릿 모드(개인정보 보호 모드)**에서 실행하는 것을 권장

# 7.2 요소 탭

현재 웹페이지를 구성하고 있는 HTML과 CSS 정보를 확인할 수 있는 곳

## 7.2.1 요소 화면

- HTML 구조 확인 및 수정 : 현재 페이지의 HTML 트리를 보여주며, 태그나 클래스 명을 더블클릭하여 직접 수정하고 결과를 즉시 확인 가능
- 중단점 설정 : 특정 요소의 하위 트리 수정, 속성 변경, 또는 노드 삭제가 일어날 때 브라우저가 실행을 멈추도록 중단점을 설정 가능

## 7.2.2 요소 정보

- 스타일/계산됨 : 요소에 적용된 스타일 소스와 최종적으로 계산된 CSS 값(크기, 패딩, 마진 등)을 확인
- 이벤트 리스너 : 현재 요소에 부착된 이벤트 목록을 보여줌
- 속성(Properties): DOM 객체가 가진 모든 속성값 확인 가능

# 7.3 소스 탭

자바스크립트 코드를 디버깅하고 중단점을 관리하는 핵심 탭

- 중단점 및 스코프 확인 : 코드가 멈춘 시점의 로컬 변수, 클로저, 전역 스코프 내의 값 확인 가능
- 호출 스택(Call Stack) : 현재 실행지점까지 어떤 경로 (실행 컨텍스트)를 거쳐왔는지 보여줌

# 7.4 네트워크 탭

웹페이지 접속 시 발생하는 모든 네트워크 통신(HTTP 요청, 웹 소켓 등)을 기록

- 필터링 : Fetch/XHR, JS, CSS 등 요청 종류별로 필터링 가능
- 디버깅 활용 : 리액트에서 동일한 API 요청이 불필요하게 두 번 발생하는 경우(`useEffect` 잘못 사용 등)를 파악할 때 유용

# 7.5 메모리 탭

애플리케이션의 메모리 누수를 찾고 성능을 최적화하는 데 사용

- 힙 스냅샷 : 특정 시점의 메모리 상태를 기록하여 객체들이 차지하는 크기를 분석
- 타임라인 할당 계측 : 시간 흐름에 따른 메모리 변화를 기록하여, 어떤 동작이 메모리 증가를 유발하는지 추적
