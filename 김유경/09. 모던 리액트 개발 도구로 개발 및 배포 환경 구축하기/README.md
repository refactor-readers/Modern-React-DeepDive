# 9.1 깃허브 액션으로 개발 환경 구축하기

- 깃 허브 액션(GitHub Actions) : 깃허브 저장소를 기반으로 소프트웨어 개발 워크플로를 자동화해 주는 도구
- CI/CD 자동화 : 테스트, 빌드, 배포 과정을 자동화하여 개발 생산성을 높임
- 유용한 액션들 !
  - calibreapp/image-actions : 이미지 파일을 자동으로 압축하여 저장소 용량을 효율적으로 관리
  - lirantal/is-website-vulnerable : 배포된 웹사이트를 방문해 사용 중인 라이브러리의 보안 취약점을 스캔
  - Lighthouse CI : 웹 성능 지표인 라이트하우스(Lighthouse) 검사를 CI 단계에서 실행하여 성능 저하를 방지

# 9.2 깃허브 Dependabot으로 보안 취약점 관리하기

- Dependabot : 프로젝트의 의존성(dependencies) 내 보안 취약점을 자동으로 감지하고 업데이트를 제안하는 도구
- 취약점 대응 : `npm audit` 등과 연계하여 실제 배포 환경에 미칠 영향을 분석하고 대응 우선 순위를 정하는 것이 중요

# 9.3 리액트 애플리케이션 배포하기

- 빠른 배포 서비스 : 서비스별로 5단계 내외의 간단한 설정으로 웹 서비스를 배포할 수 있는 클라우드 서비스를 활용
  - Vercel/Netlify : 정적 웹사이트 배포에 최적화되어 있으며, 깃허브 연동을 통한 자동 배포가 강점
  - DigitalOcean : 정적 사이트뿐만 아니라 앱 플랫폼, 가상 머신(Droplets) 등 더 다양한 클라우드 컴퓨팅

# 9.4 리액트 애플리케이션 도커라이즈하기

- 도커(Docker) : 애플리케이션을 컨테이너화하여 어떤 환경에서든 동일하게 실행할 수 있도록 보장
- Dockerfile 작성
  - CRA(Create React App) : NGINX와 같은 웹 서버를 포함하여 빌드된 정적 파일을 서빙하도록 구성합니다.
  - Next.js : 서버 사이드 렌더링 필요하므로 Node.js 런타임을 포함하여, `standalone` 옵션을 통해 이미지 크기를 최적화할 수 있음
- 이미지 배포 : 빌드된 이미지를 도커 허브(Docker Hub)나 구글 클라우드 런(Cloud Run) 등을 통해 배포하고 관리
