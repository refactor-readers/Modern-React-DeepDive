# 10.1 리액트 17 버전 살펴보기

- 버전업의 특징 : 새로운 기능 추가보다는 **점진적 업그레이드**를 가능하게 하는 구조적 변화에 집중했다
- 이벤트 위임 방식의 변경
  - 기존 리액트는 모든 이벤트를 `document`에 연결했으나, 17 버전부터는 **리액트 컴포넌트의 회상위 루트 요소(root)**에 연결
  - 이로 인해 서로 다른 버전의 리액트가 한 페이지에 섞여 있어도 이벤트 전파(Propagation)가 꼬이는 않게 되었습니다.
- 새로운 JSX Transform : `import React from 'react'` 없이도 JSX를 사용할 수 있게 되어 번들 크기를 줄이고 컴파일 효율을 높였습니다.

# 10.2 리액트 18 버전 살펴보기

- 동시성(Concurrency) 기능 : 리액트 18의 핵심은 렌더링을 쪼개고, 멈추고, 다시 시작할 수 있는 동시성 렌더링
- 새로운 훅(Hooks) 추가
  - `useTransition` 상태 업데이트를 '긴급하지 않음'으로 표시하여 UI렌더링 차단을 방지
  - `useDeferredValue` : 값의 업데이트 자체를 지연시켜 성능 최적화를 도움
  - `useId` : 서버와 클라이언트 간에 일치하는 고유한 ID를 생성
- 자동 배치(Automatic Batching)
  - 여러 상태 변경을 한 번의 렌더링으로 묶어 처리
  - 이전 버전에서는 프로미스나 타임아웃 안에서 배치가 안 되었으나, 18 버전부터는 **모든 환경에서 자동 배치**가 적용

## 10.2.4 서버 컴포넌트(Server Components)

- 개념 : 서버에서만 실행되고 브라우저로 전송되는 컴포넌트
- 장점 : 자바스크립트 번들 크기를 획기적으로 줄일 수 있으며, 서버 리소스(DB 등)에 직접 접근이 가능
